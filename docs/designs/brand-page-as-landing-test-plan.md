# Test Plan: Brand page as shareable landing
Generated by /plan-eng-review on 2026-03-18  
Branch: main | Repo: brandwall

## Affected Pages/Routes
- `/` (homepage) — Support `?country=...&brand=...`; when `brand` is present and resolves, open that brand's modal on load.
- `/[country]/[slug]` (e.g. `/georgia/spribe`) — Resolve slug to brand for that country; show wall with country selected and brand modal open; 404 when slug or country invalid/unknown.

## Key Interactions to Verify
- Visit `/georgia/spribe` → Georgia wall visible, Spribe modal open; share button copies brand URL (canonical).
- Visit `/?country=georgia&brand=spribe` → Same: Georgia selected, Spribe modal open.
- Share "this brand" from modal → Copied link is brand URL (e.g. `https://www.brandwall.online/georgia/spribe`), not `?country=georgia`.
- Visit `/georgia/nonexistent` or `/invalidcountry/any` → 404 (or redirect to country wall with not-found behavior).
- Open shared brand link in new tab → Page title and OG description show brand name and description.

## Edge Cases
- Slug with special characters in brand name (e.g. "Chateau Mukhrani" → `chateau-mukhrani`); duplicate slugs in same country (first match or stable disambiguation).
- Homepage `?brand=invalid` or `?brand=` empty → No modal opens; no error.
- Invalid country in URL → 404.

## Critical Paths
- Happy path: GET `/georgia/spribe` → 200, document title contains "Spribe", modal open.
- Deep link: GET `/?country=georgia&brand=spribe` → modal opens for Spribe.
- Not found: GET `/georgia/notabrand` → 404.
- Share: From Spribe modal, click Share → clipboard or share sheet contains brand URL.
